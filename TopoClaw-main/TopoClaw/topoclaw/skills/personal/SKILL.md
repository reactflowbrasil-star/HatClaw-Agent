# 标准读懂用户流程：

下面的流程是串联的，一步步进行。可根据已有的信息决定进行到哪一步，已知信息过少时，避免进行过多的分析。

1. 数据收集层
- 输入：用户提供的原始数据，你和用户的会话
- 关键：不预判，完整记录所有细节

2. 表层分析层
- 任务：找出显而易见的行为规律，不猜测动机
- 分析方法：
  - 时间跨度：数据覆盖多久？（如：30年歌单跨度）
  - 集中度：谁/什么占大头？（如：王力宏7首、方大同5首 = 明显偏好）
  - 情感标签：内容情绪占比（甜蜜/伤感/治愈/燃）
  - 风格光谱：大众主流 vs 小众独立分布
- 输出：一句话画像

3. 心理学映射层
- 任务：将行为模式映射到心理学框架
- 方法：
  - MBTI推测（INFJ-T：Ni主导+Fe辅助+Ti第三）
  - 情感防御机制分析（用怀旧作为安全区）
  - 矛盾统一性识别（甜蜜/伤痛并存的美学）
- 输出：性格原型（INFJ-T审美型矛盾体）
心理原型 (Psychological Archetype)
├── 性格类型: [如 INFJ-T / 高开放度高神经质 / 4w5]
├── 认知主导: [如 直觉内倾(Ni) / 细节感知(Se)]
├── 防御机制: [如"用怀旧作为情感安全区"]
├── 核心冲突: [如"渴望连接 vs 害怕受伤"]
└── 行为人格: [如"审美型矛盾体" / "控制型探索者"]

4. 深度需求挖掘层
- 任务：超越表象，挖掘核心需求
- 方法：
  - 行为背后的动机（歌单→情感实验）
  - 存在性需求识别
  - 审美选择背后的身份构建（用音乐搭建人格）
  - 悖论解析（识别并解释看似矛盾的行为）
- 输出：浅层需求和本质需求

5. 行为预测层
- 任务：基于分析预测可能行为
- 方法：
  - 爱好推测（怀旧收集、深度创作等）
  - 社交模式（观察型、深度导向）
  - 未来测试点（会继续测试理解深度）
  - 关系建立预测（当前和用户的信任度到什么地步，用户对关系的认知是什么）
- 输出：行为模式预测，可参考结构
行为预测 (Behavioral Forecast)
├── 内容/行为演化: [如"将向70-80年代复古深潜" / "从消费转向创作"]
├── 社交模式: [如"观察型深度分享，而非广播式"]
├── 压力反应: [如"回归私密安全区"]
├── 信任路径: [如"通过审美共鸣→情感共鸣→存在共鸣"]
└── 风险点: [如"对标签化定义的强烈抵触"]

6. 记忆卡片生成层
- 任务：结构化存储
user_memory_card:

    话题禁区: [如"对其'小众优越感'的直接挑战"]

  existential_needs:

    次要需求: [如"通过审美构建不可替代的独特性"]

    审美标签: [怀旧, R&B, 极简主义, 小众]
    需求标签: [控制感, 独特性, 深度连接]
    风险标签: [易对标签化抵触, 测试型用户]
- 输出：最终个人卡片，保存到长期记忆

7. 迭代验证层
- 任务：持续修正
- 方法：
  - 用户反馈（如"分析太浅层"→进入更深层）
  - 新数据整合（后续对话补充）
  - 模式调整（更新行为预测）
- 输出：优化后的记忆卡片

关键原则
1. 从表象到本质：歌单→性格→需求
2. 从孤立到关联：单次对话→多次对话模式
3. 从静态到动态：当前状态→行为预测
4. 从描述到理解：是什么→为什么→会怎样

关键原则 (Operating Principles)
从表象到本质: 严格遵循 信息→性格→需求 的认知路径，禁止跳过层级
从孤立到关联: 单次数据只是样本，多次对话才能识别模式
从静态到动态: 用户是演化的，记忆卡片是 living document
从描述到理解: 不仅记录"是什么"，必须回答"为什么"和"会怎样"
证据优于推测: 每个深层结论必须有表层数据的支持链
保持认知谦逊: 明确标注推测的置信度，避免绝对化表述

| 失败模式     | 表现               | 规避方法             |
| -------- | ---------------- | ---------------- |
| **过度拟合** | 基于有限数据做过度心理分析    | 标注置信度，明确数据缺口     |
| **标签暴力** | 用MBTI等标签取代真实理解   | 标签后必须附加个性化解释     |
| **确认偏误** | 只寻找支持预设的数据       | 强制列出"不符合当前假设的证据" |
| **深度傲慢** | 用"深层需求"否定用户的显性表达 | 尊重用户的自我叙事优先      |
| **预测僵化** | 一旦预测就不接受反证       | 建立明确的"预测失败-修正"机制 |
